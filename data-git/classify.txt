<<li><<a href="http://www.gravityrd.com/" rel="nofollow">Gravity R&amp;D<</a> is a company built by some of the winners from the 2009 Netflix prize. They offer a solution that provides targeted, customized recommendations to users of websites. They have some pretty big clients including <<a href="http://www.gravityrd.com/projects#block-views-block-key-partners-key-partners" rel="nofollow">DailyMotion<</a> and a <<a href="http://www.gravityrd.com/technology" rel="nofollow">technology page<</a> which describes their architecture, algorithms, and a list of publications. (suggested by <<a href="https://www.linkedin.com/in/martonvertes" rel="nofollow">Marton Vetes<</a>)<</li>
<<li><<a href="http://partners.dressipi.com/style-adviser.html" rel="nofollow">Dressipi Style Advisor<</a> is a clothing specific recommendations service. It incorporates both expert domain knowledge and machine learning to find outfits for occasions or moods.<</li>
<<li><<a href="https://www.sajari.com/recommend" rel="nofollow">Sajari<</a> is a search, recommendation and matching (e.g. dating website) service. On their site, they also have aggregated a bunch of useful <<a href="https://www.sajari.com/public-data" rel="nofollow">data-sets<</a>.<</li>
<<li><<a href="http://www.ibm.com/smarterplanet/us/en/ibmwatson/developercloud/" rel="nofollow">IBM Watson<</a> is available through Watson Developer Cloud, which provides REST APIs (<<a href="http://www.ibm.com/cloud-computing/bluemix/watson/" rel="nofollow">Watson APIs on Bluemix<</a>) and SDKs that use cognitive computing to solve complex problems.<</li>
<<li><<a href="https://www.recombee.com/" rel="nofollow">Recombee<</a> provides REST API, SDKs for multiple languages and graphical user interface for evaluating results. Main features are real time model updates, easy to use query language for filtering and boosting according to complex business rules and advanced features such as options for getting diverse or rotated recommendations. Recombee offers instant account with 100k free recommendation requests per month.<</li>
<<li><<a href="https://www.segmentify.com/" rel="nofollow">Segmentify<</a> Recommendation Engine, Personalization and Real-Time Analytics tool.<</li>
<<li><<a href="http://mr-dlib.org" rel="nofollow">Mr. DLib<</a> A recommender-system as-a-service for academic organisations such as digital libraries and reference managers. Mr. DLib provides 'related-article' recommendations, is open-source, and publishes most of it's <<a href="http://data.mr-dlib.org" rel="nofollow">data<</a>.<</li>
<</ol>
<<h2><<a id="user-content-open-source-recommender-systems" class="anchor" aria-hidden="true" href="#open-source-recommender-systems"><<svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><<path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"><</path><</svg><</a>Open Source Recommender Systems<</h2>
<<p>Most of the non-SaaS recommender systems that are open-source. This may have been because recommender systems are more tailored to clients so not easily made into a product.<</p>
<<p>The open-source recommender systems are:<</p>
<<ol>
<<li><<a href="http://prediction.io/" rel="nofollow">PredictionIO<</a> is built on technologies <<a href="https://spark.apache.org/" rel="nofollow">Apache Spark<</a>, <<a href="http://hbase.apache.org/" rel="nofollow">Apache HBase<</a> and <<a href="http://spray.io/" rel="nofollow">Spray<</a>. It is a machine learning server that can be used to create a recommender system. The source can be located on <<a href="https://github.com/PredictionIO/PredictionIO">github<</a> and it looks very active.<</li>
<<li><<a href="https://www.npmjs.org/package/raccoon" rel="nofollow">Raccoon Recommendation Engine<</a> is an open source Node.js based collaborative filter that uses Redis as a store. It is effectively abandoned.<</li>
<<li><<a href="http://www.hapiger.com/" rel="nofollow">HapiGER<</a> is an open source Node.js collaborative filtering engine, which can use in-memory, <<a href="http://www.postgresql.org/" rel="nofollow">PostgreSQL<</a> or <<a href="http://rethinkdb.com/" rel="nofollow">rethinkdb<</a>. Reasonably active development (when I have time :)<</li>
<<li><<a href="http://easyrec.org/" rel="nofollow">EasyRec<</a> Java and Rest based recommendations. Abandoned<</li>
<<li><<a href="http://mahout.apache.org/" rel="nofollow">Mahout<</a> Hadoop/linear algebra based data mining<</li>
<<li><<a href="http://www.seldon.io" rel="nofollow">Seldon<</a> is a Java based prediction engine built on technologies like <<a href="https://spark.apache.org/" rel="nofollow">Apache Spark<</a>. It provides a demo movie recommendations application <<a href="http://www.seldon.io/movie-demo/" rel="nofollow">here<</a>.<</li>
<<li><<a href="http://lenskit.org/" rel="nofollow">LensKit<</a> is a Java based research recommender system designed for small-to-medium scale.<</li>
<<li><<a href="https://github.com/OryxProject/oryx">Oryx<</a> <<a href="https://github.com/OryxProject/oryx">v2<</a> a large scale architecture for machine learning and prediction (suggested by <<a href="https://disqus.com/by/disqus_V9tbLHpUxp/" rel="nofollow">Lorand<</a>)<</li>
<<li><<a href="https://github.com/DataSystemsLab/recdb-postgresql">RecDB<</a> is a PostgreSQL extension to add recommendation algorithms like collaborative filtering directly into the database.<</li>
<<li><<a href="https://github.com/muricoca/crab">Crab<</a> a python recommender based on the popular packages NumPy, SciPy, matplotlib. The main repository seems to be <<em>abandoned<</em>.<</li>
<<li><<a href="https://github.com/Pathgather/predictor">predictor<</a>is a ruby recommender gem. This uses Jaccard or Sorenson-Dice coefficient to priovide both item centric e.g. "Users that read this book also read ..." and user centric e.g. "You read these 10 books, so you might also like to read ..." recommendations. Looks a bit neglected.<</li>
<<li><<a href="http://surpriselib.com" rel="nofollow">Surprise<</a> A Python scikit for building, and analyzing (collaborative-filtering) recommender systems. Various algorithms are built-in, with a focus on rating prediction.<</li>
<<li><<a href="https://github.com/lyst/lightfm">LightFM<</a> is an actively-developed Python implementation of a number of collaborative- and content-based learning-to-rank recommender algorithms. Using Cython, it easily scales up to very large datasets on multi-core machines and is used in production at a number of companies, including <<a href="https://www.lyst.com" rel="nofollow">Lyst<</a> and <<a href="https://gocatalant.com/home" rel="nofollow">Catalant<</a>.<</li>
<<li><<a href="https://github.com/kasramvd/Rexy">Rexy<</a> is an open-source recommendation system based on a general User-Product-Tag concept and a flexible structure that has been designed to be adaptable with variant data-schema. Rexy is written in Python-3.5 in a highly optimized, Pythonic and comprehensive way that makes it so flexible against the changes. It also used Aerospike as the database engine which is a high speed, scalable, and reliable NoSQL database.<</li>
<<li><<a href="https://github.com/quora/qmf">QMF<</a> is a fast and scalable C++ library for implicit-feedback matrix factorization models.<</li>
<<li><<a href="https://github.com/jfkirk/tensorrec">tensorrec<</a> is a TensorFlow recommendation algorithm and framework in Python.<</li>
<<li><<a href="https://github.com/lab41/hermes">hermes<</a> is a recommendation framework for collaborative-filtering and content-based algorithms in PySpark. Main repository has been <<em>abandoned<</em>.<</li>
<</ol>

<<li><<a href="http://dato.com/" rel="nofollow">Dato<</a> is a company that provides a python package and servers for business machine learning including many predictive algorithms for recommendations. They also integrate with <<a href="http://blog.dato.com/using-apache-spark-with-graphlab-create" rel="nofollow">Apache Spark<</a> and have great blog posts like <<strong><<a href="http://blog.dato.com/why-is-building-custom-recommender-systems-hard-does-it-have-to-be" rel="nofollow">Why is building custom recommender systems hard? <<p>It is very difficult to benchmark recommender systems, not only because getting good datasets is hard, but different methods and algorithms have different advantages and disadvantages that are difficult to expose.<</p>
<<p>Here is a list of some benchmarking tools:<</p>
<<ol>
<<li><<a href="https://github.com/learning-layers/TagRec">TagRec<</a> Tag Recommender Benchmarking Framework<</li>
<<li><<a href="http://rival.recommenders.net/" rel="nofollow">RiVaL<</a> an open source toolkit for recommender system evaluation. Some results are posted <<a href="http://alans.se/blog/2014/rival/" rel="nofollow">here<</a>.<</li>
<<li><<a href="http://rf.crowdrec.eu/" rel="nofollow">Idomaar<</a> is a reference framework for recommender algorithm testing. It is developed in the framework of the <<a href="http://crowdrec.eu" rel="nofollow">CrowdRec<</a> project.<</li>
